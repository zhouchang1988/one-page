# 技能五子棋 (Skills Five)

> 类型：游戏

一个带有特殊技能的五子棋游戏，支持双人对战和人机对战，基于纯前端技术实现，无需后端服务器。

## 🎮 游戏特色

### 基础规则
- **棋盘大小**：15×15 标准五子棋棋盘
- **下棋位置**：棋子下在线条的交叉点上
- **胜利条件**：任意方向（横、竖、斜）连成5个或以上棋子即获胜
- **对战模式**：双人对战（PvP）或人机对战（PvE）

### 游戏模式

#### 👥 双人对战
两位玩家在同一设备上轮流下棋，黑棋先行。

#### 🤖 人机对战
与 AI 对手对战，AI 执白棋。AI 采用评分算法，综合考虑进攻与防守：
- 评估每个空位的进攻价值和防守价值
- 识别活四、冲四、活三、眠三等棋型
- 只评估有邻居的位置以优化性能
- 防守权重略高于进攻

### 特殊技能系统

每位玩家拥有4个独特技能，增加了游戏的策略性和趣味性：

#### 1. ⚡ 飞沙走石
- **效果**：选择并移除对方棋盘上的一个棋子
- **冷却时间**：3回合
- **使用方法**：点击技能按钮激活，对方棋子会高亮显示，点击目标棋子即可移除

#### 2. 💧 静如止水
- **效果**：让对方暂停一回合（对方无法下棋）
- **冷却时间**：3回合
- **使用方法**：点击技能按钮立即生效

#### 3. ⏰ 时光倒流
- **效果**：退回到上一回合的状态
- **冷却时间**：3回合
- **使用方法**：点击技能按钮立即生效

#### 4. 💪 力拔山兮
- **效果**：直接获胜
- **使用限制**：每局游戏只能使用一次，且需要双方都至少下过一子后才可使用
- **使用方法**：点击技能按钮，确认后触发炸飞特效——棋盘震动、所有棋子飞散——最终宣告获胜

## 🏗️ 项目架构

### 技术栈
- **前端框架**：纯 JavaScript (ES6+)
- **样式**：CSS3（使用 Flexbox 和 Grid 布局）
- **响应式设计**：支持桌面和移动设备

### 文件结构
```
skill-five/
├── index.html      # 主页面结构
├── style.css       # 样式文件
├── game.js         # 游戏核心逻辑
├── app.js          # 前端交互逻辑
└── README.md       # 项目文档
```

### 核心模块

#### game.js - 游戏逻辑类
负责游戏的核心规则和状态管理：
- `GomokuGame` 类：管理游戏状态、棋盘、玩家回合
- 棋盘初始化和落子逻辑
- 胜利条件检测（横、竖、斜四个方向）
- 技能系统实现（冷却管理、效果执行）
- 历史记录管理（用于时光倒流）
- AI 对手逻辑（评分算法、棋型识别、落子决策）

#### app.js - 前端交互
处理用户界面交互：
- 棋盘渲染和更新
- 技能按钮状态管理
- 玩家回合切换显示
- 技能激活和目标选择
- 游戏模式选择（PvP / PvE）
- 弹窗管理（游戏说明、重新开始确认、力拔山兮确认）
- 力拔山兮炸飞特效动画
- AI 回合自动落子

#### style.css - 视觉设计
提供美观的用户界面：
- 渐变背景和现代化UI设计
- 棋盘交叉点显示效果
- 当前玩家高亮动画
- 技能按钮样式和冷却显示
- 力拔山兮震动与炸飞动画
- 弹窗组件样式
- 游戏模式选择界面
- 响应式布局适配

### 游戏流程

1. **模式选择**：启动时弹出模式选择，选择双人对战或人机对战
2. **初始化**：创建15×15棋盘，黑棋先行
3. **玩家回合**：
   - 正常下棋：点击交叉点落子
   - 使用技能：点击技能按钮，根据技能类型执行相应操作
4. **回合切换**：自动切换到对方玩家，减少技能冷却时间
5. **AI回合**（人机模式）：AI 自动评估并落子
6. **胜利判定**：每次落子后检查是否满足胜利条件
7. **游戏结束**：显示获胜玩家，可重新开始

## 🚀 使用方式

直接在浏览器中打开 `index.html` 即可开始游戏。

```bash
# macOS
open index.html

# Linux
xdg-open index.html

# Windows
start index.html
```

## 🎯 游戏策略建议

1. **飞沙走石**：适合在对方即将连成5子时使用，破坏对方的进攻节奏
2. **静如止水**：在自己进攻关键期使用，为自己争取额外一回合
3. **时光倒流**：在失误后立即使用，挽回不利局面
4. **力拔山兮**：作为终极技能，在关键时刻使用，确保胜利。注意需要双方各至少下一子后才可使用
